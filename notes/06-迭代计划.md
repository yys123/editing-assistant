# 06 迭代计划生成

**文件**: `backend/services/analyzer.py`
**涉及函数**: `generate_plan_from_gap()`（主用）、`generate_iteration_plan()`（旧版）

## System Prompt

同质量审评。

---

## 提示词 1：基于审核结果生成迭代计划（主用）

**函数**: `generate_plan_from_gap()`
**说明**: 综合章节质量问题和用户需求差距，按词条实际章节顺序生成具体的迭代任务。每条任务的 description 包含现状、目标、依据、修订要点，可直接驱动后续的内容生成。

```
基于以下【{disease}】词条经过审核的质量问题和用户需求差距，生成按章节结构组织的内容迭代计划。

## 词条实际章节结构（请严格按此顺序和命名输出迭代任务）
{structure_text}

## 各章节质量问题（已过滤被排除的条目）
{issues_text}
  ← 每条标注：【已确认】/【AI识别】 + 严重程度 + 是否重点内容

## 各章节用户需求差距（已过滤被排除的条目）
{needs_text}
  ← 每条标注：覆盖状态 + 用户提问频次 + 修订建议

## 章节粒度规则（决定迭代任务的section字段精度）
- 「基础知识」「预后」「预防」等篇幅较小的章节：section直接使用1级标题
- 「鉴别诊断」：问题简单时使用"鉴别诊断"，复杂时细分到2级
- 「诊断」「治疗」等有复杂子目录的章节：优先细化到2级或3级子章节
- section命名必须与实际章节结构一致，使用 " > " 连接层级

## 优先级规则
- P0：【已确认】的高优问题，内容错误，重点内容缺失，高频未覆盖需求（>100次）
- P1：【已确认】的中优问题，内容陈旧，结构问题，中频需求（50-100次）；或【AI识别】的高优问题
- P2：非重点内容完善，精炼优化，低频需求，【AI识别】未确认的一般问题

## description字段撰写要求（关键）
description必须足够详细具体，使后续生成稿件的AI能直接依据它输出高质量修订内容。每条description需包含：
1. 【现状】当前该子章节存在什么具体问题
2. 【目标】需要补充或修改成什么
3. 【依据】基于哪些质量问题或用户需求
4. 【修订要点】建议的具体修改方式

示例description格式：
"当前【实验室检查】部分仅列出血常规、尿常规等基础检查，缺少特异性诊断指标（用户频繁提问约85次，审核标记高优）。需补充：①特异性血清学标志物的检测意义及正常参考值；②各项指标的诊断阈值及临床解读方法；③不同病程阶段检查项目的选择时机。建议在现有检查项目列表后新增'特异性指标'子段落，格式与现有内容保持一致。"

source取值：quality_eval/user_needs/both
注意：rejected的条目不纳入计划；同一子章节的多个问题可合并为一条；按章节顺序输出。
```

**输出格式**: JSON，含 `gap_items` 数组

## 提示词 2：旧版迭代计划（基于四维度评估）

**函数**: `generate_iteration_plan()`
**说明**: 基于旧版四维度质量评估报告和需求分析生成优先级计划。已较少使用。

```
基于以下【{disease}】经专家审核的质量评估报告和用户需求分析，生成内容迭代优先级计划。

## 专家审核后的质量评估
{quality_summary}
  ← 四维度评分 + 各子类问题列表 + 审核意见

## 用户需求分析（来自真实Q&A）
{needs_summary}

请以JSON格式输出迭代优先级计划：
{
  "gap_items": [
    {
      "priority": "P0/P1/P2",
      "section": "受影响的章节名称",
      "description": "具体需要补充/修改/重构的内容描述",
      "source": "quality_eval/user_needs/both",
      "qa_frequency": 987
    }
  ]
}

优先级规则：
- P0：高优先（内容错误、重点内容缺失、高频需求>100次且未覆盖）
- P1：中优先（内容陈旧、结构不合理、中频需求50-100次）
- P2：低优先（非重点内容补充、精炼流畅类问题、低频需求）

注意：被专家标记为rejected的问题已从评估中移除。专家补充或确认的问题应优先考虑。
```
