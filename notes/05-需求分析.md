# 05 需求分析

**文件**: `backend/services/analyzer.py`
**涉及函数**: `analyze_user_needs()`、`_classify_batch()`、`_merge_need_clusters()`、`map_needs_to_sections()`、`_analyze_needs_gap_single()`、`suggest_unmapped_placements()`、`analyze_gap()`

## System Prompt

同质量审评：
```
你是一位资深临床医学编辑，专注于面向临床医生的循证医学内容评估与改进。
请严格按照要求的JSON格式输出，不要输出任何其他内容。
```

---

## 提示词 1：用户需求聚类（旧版）

**函数**: `analyze_user_needs()`
**说明**: 对全部 QA 数据（取前200条）进行需求主题聚类。

```
以下是临床医生关于【{disease}】的真实提问（共{len(qa_items)}条）。

{qa_text}

请分析这些提问，识别出典型需求类型并聚类，以JSON格式输出：
{
  "disease": "{disease}",
  "total_qa_count": {len(qa_items)},
  "clusters": [
    {
      "topic": "需求主题（简洁描述）",
      "frequency": <该类问题估计数量>,
      "representative_questions": ["典型问题1", "典型问题2"],
      "covered_in_kb": <true或false>,
      "coverage_notes": "覆盖情况说明"
    }
  ]
}

要求：
- 聚类5-10个主要需求类型
- 按频次从高到低排列
- representative_questions每类取2-3个最有代表性的
```

## 提示词 2：QA 批量分类

**函数**: `_classify_batch()`
**说明**: 对一批 QA 数据（最多200条/批）进行聚类，支持大量数据分批处理。

```
以下是临床医生关于【{disease}】的真实提问（本批{len(batch_items)}条，总计{total_count}条）。

{qa_text}

请对这些提问进行需求分析与分类，识别出典型需求类型并聚类，以JSON格式输出：
{
  "clusters": [
    {
      "topic": "需求主题（简洁描述）",
      "frequency": <该类问题估计数量>,
      "representative_questions": ["典型问题1", "典型问题2", "典型问题3"],
      "covered_in_kb": <true或false>,
      "coverage_notes": "覆盖情况说明"
    }
  ]
}

要求：
- 聚类5-10个主要需求类型，按频次从高到低排列
- representative_questions每类取2-3个最有代表性的
- covered_in_kb根据问题本身判断，不确定时填false
```

---

## 提示词 3：聚类合并

**函数**: `_merge_need_clusters()`
**说明**: 将多批聚类结果合并去重。

```
以下是对【{disease}】用户Q&A分批聚类后得到的多组需求类型（来自不同批次，可能有重复或相似主题）。

{clusters_text}

请将相同或高度相似的需求主题合并，汇总频次，保留最有代表性的问题，以JSON格式输出最终聚类结果：
{
  "clusters": [ ... ]
}

要求：
- 合并相同/相近主题，最终输出5-12个主要需求类型
- frequency为各批次同类需求频次之和（估算）
- 按最终频次从高到低排列
- covered_in_kb：若任何批次标记为false则取false
```

---

## 提示词 4：需求映射到章节

**函数**: `map_needs_to_sections()`
**说明**: 将需求聚类映射到词条章节结构，判断各章节与哪些需求最相关。

```
以下是【{disease}】知识库词条的章节结构和用户需求聚类。

## 章节结构
{sections_text}

## 用户需求聚类
{clusters_text}

请判断每个章节与哪些需求类型最相关，以JSON格式输出：
{
  "mappings": [
    {
      "section_id": "章节ID（原样复制）",
      "section_heading": "章节标题",
      "cluster_topics": ["需求主题1", "需求主题2"],
      "relevance": "high"
    }
  ]
}

relevance取值：
- high：该章节是满足这些需求的核心章节
- medium：有一定相关性
- low：相关性较弱，或该章节无对应需求

每个章节必须出现在mappings中（即使cluster_topics为空也要包含）。
```

## 提示词 5：章节需求覆盖评估

**函数**: `_analyze_needs_gap_single()`
**说明**: 逐章节评估对映射需求的覆盖程度。支持长章节分块。

```
请分析【{disease}】词条「{section.heading}」章节对以下用户需求的覆盖情况。

## 章节内容
{section.content[:15000]}

## 需要评估的用户需求（共{len(mapped_clusters)}类）
{clusters_text}

{cross_section_block}  ← 跨章节提示：某些需求同时映射到其他章节

请对每类用户需求，评估该章节内容的满足程度，以JSON格式输出：
{
  "coverage_assessment": "对本章节整体覆盖情况的简评",
  "need_coverages": [
    {
      "topic": "需求主题（与上方完全一致）",
      "coverage_level": "full/partial/missing",
      "qa_frequency": 50,
      "representative_questions": ["代表性问题1"],
      "revision_suggestion": ""
    }
  ]
}

coverage_level取值规则：
- "full"：章节已有完整内容，能清晰回答该类需求
- "partial"：有相关内容但不够完整
- "missing"：章节基本未涉及该类需求

revision_suggestion：full时留空，partial/missing时说明需要补充什么
```

## 提示词 6：未映射需求放置建议

**函数**: `suggest_unmapped_placements()`
**说明**: 对未被任何章节映射的需求，建议放置位置。

```
以下是【{disease}】词条当前章节结构，以及一批用户高频提问的需求主题，这些需求主题目前在词条中没有找到对应的章节内容。

## 当前词条章节大纲
{section_outline}

## 未覆盖的用户需求
{clusters_text}

请对每类未覆盖的需求，给出具体的内容补充建议：
1. 是否建议新增独立章节
2. 或在现有某个章节中补充

要求：
- 建议须具体可操作，不超过150字
- 优先融入现有章节；若内容独立性强，再建议新增章节
- 若建议新增章节，说明插入位置
```

## 提示词 7：综合差距分析（旧版）

**函数**: `analyze_gap()`
**说明**: 单次调用完成需求聚类 + 差距识别（用于较少 QA 数据的场景）。

```
基于以下【{disease}】词条的章节质量分析和用户Q&A数据，识别需求差距。

## 章节质量分析摘要
{issues_summary}

## 用户Q&A数据（共{len(qa_items)}条）
{qa_text}

请以JSON格式输出需求差距分析：
{
  "total_qa_count": ...,
  "clusters": [ ... ],
  "unmet_needs": [
    {
      "priority": "P0/P1/P2",
      "section": "相关章节名",
      "description": "具体差距描述和改进方向",
      "source": "quality_eval/user_needs/both",
      "qa_frequency": 80
    }
  ],
  "optimization_suggestions": ["优化建议"]
}

优先级规则：
- P0: 高频需求(>100次)未覆盖，或内容高优先问题
- P1: 中频需求(50-100次)，或内容中优先问题
- P2: 低频需求或内容低优先改进
```
