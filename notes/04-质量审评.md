## 提示词 1：整体质量评估（旧版四维度）

**函数**: `evaluate_article_quality()`
**说明**: 对整篇词条进行四维度评估（取前8000字），输出各维度评分和问题列表。

```
请对以下【{disease}】疾病知识库词条内容进行专业质量评估，严格按照以下四个维度标准逐项分析。

词条内容：
{article_text[:8000]}

## 评估维度说明（依据内容质量审评标准3.0）

**维度一：内容全面**
- 重点内容（定义/概述、典型临床表现、主要辅助检查、诊断标准/诊断、治疗）：缺失须有权威数据源支撑才算问题
- 非重点内容（分类/分型、发病机制、病理生理、解剖、少见临床特征、预后、预防等）：需个性化判断
- 数据源缺失：关键内容无权威来源，每缺1篇扣1分
- 合格阈值：重点扣分＜8分/万字，或累积＜15分/万字；优秀：重点＜4，或累积＜8
- 扣分参考：重点内容重大缺失5分，影响临床使用3分，一般1分；非重点1分；数据源每缺1篇1分

**维度二：内容准确**
- 内容错误：文本性错误影响句意（如药物剂量数字误写）：扣1分/处
- 内容陈旧：引用过时指南或旧版数据：扣1分/处
- 内容合理性（判定标准：是否影响用户理解、是否符合临床思维，文献不一致时保留的内容应自洽清楚）
- 合格阈值：重点扣分＜8分/万字，或累积＜15分/万字；优秀：重点＜4，或累积＜8

**维度三：结构合理**
- 临床思维：章节顺序应结合具体疾病特点灵活判断
- 整合逻辑：分类/分型/分期须放在正确字段；标题须能概括下属全部内容
- 合格阈值：累积＜15分/万字；优秀：累积＜5分/万字

**维度四：内容精炼流畅**
- 同类错误多次出现计为1处，每处扣0.5分
- 合格阈值：累积＜20分/万字；优秀：累积＜10分/万字
- 扣分项：重复性文本、研究性内容未精简、图表问题、英文缩写问题、标题问题、格式不一致等
```

**输出格式**: JSON，含 `overall_score`、`summary`、`dim_one` ~ `dim_four`（各含 score + 问题列表）

## 提示词 2：章节质量分析（分块）

**函数**: `_analyze_section_chunk()`
**说明**: 对单个章节（或长章节的一个分块）进行五类问题分析。注入了质量标准和内容规范。

```
请对【{disease}】知识库词条的以下章节进行质量分析。{chunk_note}章节内容已包含其下级子章节的内容，请综合考虑整体。

## 章节标题
{chunk_section.heading}

## 章节内容（含子章节）
{chunk_section.content}

{outline_block}  ← 完整词条章节大纲 + 关键规则：不得将其他章节职责范围的内容标记为本章节缺失

## 内容质量审评标准
{quality_standard[:3000]}

## 内容要求规范
{content_spec[:2000]}

{ref_block}  ← 参考文献摘录

请从以下五类问题角度分析：

issue_type取值（对应审评维度）：
- missing_content: 【维度一·内容全面】内容缺失
- accuracy: 【维度二·内容准确】内容不准确或有误
- outdated: 【维度二·内容准确】内容陈旧
- structure: 【维度三·结构合理】结构/逻辑问题
- style: 【维度四·内容精炼流畅】语言/格式/重复问题

deduction_score扣分参考（实际扣分值，不做万字换算）：
- missing_content重点内容：1-5分；非重点1分；数据源每缺1篇1分
- accuracy内容错误：1分；内容合理性问题——多源融合混乱：3分；分型未说明特点/优缺点：1分；剂量差异/特殊用法未列举：1分；表格文字冗余重叠：2分
- outdated内容陈旧：1分
- structure临床思维大问题：3分；小结构调整：1分
  ⚠️ 结构评估重要原则：子级目录是参考框架，并非强制要求。只有内容放错位置或层级逻辑混乱才应标记。
- style同类问题每处0.5分

is_key_content说明：重点内容包括定义/概述、典型临床表现、主要辅助检查、诊断标准/诊断、治疗
```

**输出格式**: JSON，含 `issues` 数组

## 提示词 3：分块问题合并

**函数**: `_merge_chunk_issues()`
**说明**: 将多个分块分析的问题合并去重。

```
以下是【{disease}】词条「{section_heading}」章节{total_chunks}段分析结果的汇总。
多段分析可能对同一问题重复报告或从不同角度描述同一问题。

## 所有分段发现的问题（共{len(all_issues)}项）
{issues_text}

请对以上问题进行去重和合并，规则：
1. **合并**：如果两条问题描述的是同一个内容点，合并为一条，保留最准确的描述和扣分
2. **保留**：不同内容点的问题即使类型相同也要保留
3. **扣分**：合并后的扣分取原问题中的最高值
4. **示例保留**：合并时保留所有不重复的示例

以JSON格式输出：
{
  "merge_summary": "合并说明",
  "issues": [ ... ]
}
```

## 提示词 5：问题二次校验

**函数**: `_verify_section_issues()`
**说明**: 对第一次分析结果进行二次核查，过滤误报、优化描述。

```
请对【{disease}】知识库词条「{section.heading}」章节的质量分析结果进行二次校验与优化。

## 章节原文内容
{section.content[:15000]}

## 第一次分析识别到的问题（共{len(first_pass_issues)}项）
{issues_text}

## 内容质量审评标准
{quality_standard[:2000]}

## 内容要求规范
{content_spec[:1500]}

请对上述每个问题逐一核查：
1. **核实**：对照章节原文，确认该问题是否真实存在，有无被原文实际内容覆盖
2. **优化**：若问题描述不够精准，基于原文修正描述并补充具体原文依据
3. **删除**：若某问题在原文中已有明确覆盖，或描述明显有误，则移除；特别地，若structure类问题指向的是"缺少某子级目录"，但该章节内容本身不涉及该内容，则应删除
4. **合并**：若多个问题高度重叠，合并为一条更准确的描述

若所有问题核验后均属实且无需修改，原样输出并说明「问题属实，无需调整」。
```

**输出格式**: JSON，含 `verification_summary` 和 `issues` 数组
