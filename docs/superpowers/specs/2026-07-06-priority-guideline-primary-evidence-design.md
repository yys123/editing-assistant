# 重点指南主证据约束设计

## 目标

让用户在内容质量评审和 AI 整合中设置的“重点指南”真正成为 AI 判断、引用和修订时的核心依据，而不是普通 prompt 提醒。只要重点指南覆盖相关事实、推荐、诊断标准、治疗方案、分期、剂量、疗程、适应证或禁忌证，系统应优先采用重点指南；普通参考资料只能补充重点指南未覆盖的内容。

## 背景

当前流程已经支持用户设置重点指南：

- 质量评审前端将本章节重点指南作为 `priority_reference_texts` 传给后端。
- AI 整合前端将重点指南编号作为 `priority_reference_ids` 传给后端。
- 后端 prompt 中已有“重点指南优先”的规则文字。

现有缺口在于：重点指南还没有形成完整闭环。AI 整合中“重点指南清单”主要是文件名列表，普通参考资料和重点指南仍混在同一个证据选择与引用流程中；质量评审中虽然有重点指南片段，但缺少输出后校验和前端可见的“是否使用重点指南”状态。结果就是模型看见了规则，但在长上下文、多资料、资料冲突或用户问题较宽泛时，仍可能把普通资料当主体依据。

## 范围

包含：

- 将“重点指南”定义为主证据来源。
- AI 整合构造独立的重点指南证据区，而不只是重点指南文件名清单。
- 质量评审继续优先注入重点指南片段，并强化 `guideline_evidence` 的重点指南使用要求。
- 后端对 AI 输出做轻量校验，识别重点指南未使用、证据不足或可能被普通资料覆盖的情况。
- 前端在结果中展示重点指南使用状态。
- 保持旧数据兼容。

不包含：

- 自动判断哪篇资料应被设为重点指南。
- 自动从外部网络检索最新版指南。
- 自动改写原词条并回写知识库。
- 对每条医学结论做完整自然语言推理级冲突判定。

## 推荐方案

采用“主证据闭环”：

1. 前端选择重点指南。
2. 后端把重点指南转成独立主证据上下文。
3. Prompt 明确主证据优先规则。
4. AI 输出必须说明重点指南使用情况。
5. 后端检查输出是否引用或使用了重点指南。
6. 前端展示本次重点指南使用状态。

这样把“是否参考重点指南”从模型自由发挥，升级为可见、可检查、可测试的系统行为。

## 语义定义

重点指南的系统语义改为：

> 重点指南 = primary evidence sources。若重点指南覆盖相关问题，AI 必须优先采用重点指南结论；普通参考资料只能用于补充重点指南未覆盖的内容。若普通参考资料与重点指南冲突，AI 应说明冲突，并采用重点指南结论，除非用户明确要求比较不同资料。

前端文案可以继续叫“重点指南”，后端和测试中建议使用 `primary evidence` 或 `priority guideline` 表达该语义。

## AI 整合数据流

现有请求字段可继续使用：

```py
class AiIntegrationRequest(BaseModel):
    disease: str
    user_request: str
    original_content: str = ""
    reference_inputs: List[ReferenceInput] = []
    priority_reference_ids: List[int] = []
```

后端构造上下文时拆成两个区：

- `primary_reference_text`：重点指南证据摘录区。
- `supplementary_reference_text`：普通参考资料证据区。

重点指南证据摘录区规则：

- 在 prompt 中放在普通参考资料前。
- 标题明确写为“重点指南主证据区”。
- 每个重点指南至少保留若干高相关片段。
- 如果相关性检索没有命中，仍保留该重点指南开头或核心片段，避免重点指南完全消失。
- 证据块继续保留现有引用标记，如 `[2-15]`、`[3]`。

普通参考资料区规则：

- 排除已经进入主证据区的重复片段。
- 明确标注“仅用于补充重点指南未覆盖内容”。
- 引用规则不变。

## AI 整合输出契约

AI 整合 prompt 应要求固定包含：

```md
## 修订后正文
...

## 重点指南使用情况
- 已采用：参考数据源 2：xxx指南，用于诊断标准/治疗推荐，引用：[2-15]
- 未覆盖：重点指南未提供随访频率，使用参考数据源 4 补充，引用：[4-8]

## 修改说明
- ...
```

后端可以先解析 Markdown 区块，不必一次性改为强 JSON。后续如果要进一步稳定，可再升级为结构化 JSON。

扩展响应模型：

```py
class PriorityGuidelineUsage(BaseModel):
    status: str  # used / not_covered / not_used / unknown
    used_sources: List[str] = []
    warnings: List[str] = []

class AiIntegrationResponse(BaseModel):
    answer: str
    revision_text: str = ""
    change_summary: List[str] = []
    references_used: List[str] = []
    reference_anchors: List[ReferenceAnchor] = []
    priority_guideline_usage: Optional[PriorityGuidelineUsage] = None
```

兼容规则：

- 老前端没有读取 `priority_guideline_usage` 时不受影响。
- AI 没有按契约输出“重点指南使用情况”时，后端通过引用标记做兜底判断。
- 没有设置重点指南时，`priority_guideline_usage.status = "not_configured"` 或返回空。

## AI 整合后端校验

生成后做轻量校验：

- 如果设置了重点指南，且 `answer` / `revision_text` 没有引用任何重点指南编号，标记 `not_used`。
- 如果设置了重点指南，且输出明确说重点指南未覆盖，标记 `not_covered`，并保留说明。
- 如果输出引用了重点指南，标记 `used`。
- 如果无法判断，标记 `unknown`，前端提示需人工确认。

第一版不做复杂医学语义冲突检测，只基于重点指南编号、引用标记和 AI 的“重点指南使用情况”区块判断。这能以较低风险提供可观察性。

## 质量评审数据流

质量评审已有 `priority_reference_texts`，保留该接口。需要强化三个点：

1. `_build_priority_reference_block()` 明确输出“重点指南主证据区”，并说明它是主判断依据。
2. `_analyze_section_with_reference_block()` 中的 `guideline_evidence` 要求增加优先级：若重点指南区覆盖该问题，必须优先引用重点指南；普通资料只补充。
3. `_verify_section_issues()` 或现有核验逻辑中增加检查：对 `missing_content`、`accuracy`、`outdated` 问题，如果设置了重点指南但没有任何重点指南依据，要求 AI 判断“重点指南未覆盖”或补全重点指南依据。

## 质量评审输出处理

对每个问题保留现有 `guideline_evidence` 字段，不新增复杂结构。后端可增加轻量派生字段或前端计算：

- `has_priority_guideline_evidence`
- `priority_guideline_status`

判断规则：

- `guideline_evidence.source` 对应重点指南来源编号，则为 `used`。
- 没有重点指南依据但问题类型依赖指南判断，则为 `unknown` 或 `not_used`。
- 若问题描述或核验结果明确重点指南未覆盖，则为 `not_covered`。

第一版可以只在前端展示 AI 整合的使用状态；质量评审先把重点指南依据显示得更明显，后续再做每条问题状态。

## 前端设计

AI 整合结果区域增加一个紧凑状态：

- `已优先采用重点指南`
- `重点指南未覆盖，已使用普通资料补充`
- `未检测到重点指南引用，建议重新生成或人工核查`
- `无法判断重点指南使用情况`

状态应靠近回答结果或历史记录头部，不需要打断用户阅读。

质量评审问题卡片中，如果 `guideline_evidence` 来自重点指南，可在依据旁显示 `重点指南依据`。如果设置了重点指南但问题没有重点指南依据，可以不在第一版显示警告，避免让界面过度噪声化。

## 错误处理

- 未设置重点指南时，流程与现在一致。
- 重点指南全文过长时，使用相关片段加兜底开头片段，不阻塞请求。
- AI 没有输出“重点指南使用情况”时，后端用引用标记兜底。
- 校验无法判断时，不让请求失败，只返回 `unknown` 和 warning。
- 重点指南引用状态只作为辅助质量信号，不自动删除 AI 回答。

## 测试

后端测试：

- AI 整合设置 `priority_reference_ids` 时，prompt 中包含独立“重点指南主证据区”。
- 重点指南片段出现在普通资料之前。
- 重点指南无相关命中时，仍有兜底片段进入 prompt。
- AI 回答引用重点指南编号时，`priority_guideline_usage.status = "used"`。
- AI 回答未引用重点指南编号时，`status = "not_used"` 并带 warning。
- 无重点指南时不改变旧行为。
- 质量评审 prompt 中保留重点指南主证据规则。

前端测试：

- AI 整合记录能保存并展示 `priorityGuidelineUsage`。
- `used`、`not_covered`、`not_used`、`unknown` 显示不同状态文案。
- 老记录没有该字段时正常展示。
- 重点指南依据在质量评审问题中能被标识出来。

## 实施顺序

1. 后端新增重点指南使用状态模型，并扩展 AI 整合响应。
2. 将 AI 整合中的“重点指南清单”升级为“重点指南主证据区”。
3. 增加 AI 整合输出契约和后端兜底校验。
4. 扩展前端类型和历史记录保存逻辑。
5. 在 AI 整合结果中展示重点指南使用状态。
6. 强化质量评审重点指南 prompt 和核验要求。
7. 增加后端和前端测试。

## 验收标准

- 用户设置重点指南后，后端发送给模型的 prompt 中能看到独立主证据区。
- AI 整合结果能告诉用户本次是否使用了重点指南。
- 如果 AI 没有引用重点指南，用户能看到明确提示，而不是以为设置已生效。
- 内容质量评审中，依赖指南判断的问题优先保留重点指南原文依据。
- 未设置重点指南的工作流与现有行为保持兼容。

## 后续增强

- 引入更强的医学主题匹配，判断普通资料结论是否覆盖了重点指南覆盖领域。
- 将 AI 整合输出从 Markdown 区块升级为严格 JSON。
- 对参考资料自动识别指南年份、机构和证据等级，辅助用户选择重点指南。
- 在质量评审每条问题上展示更细的重点指南覆盖状态。
