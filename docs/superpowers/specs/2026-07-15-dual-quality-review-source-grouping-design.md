# 内容质量双重评审分组展示设计

## 目标

在“内容质量评审”模块中保留当前基于质量标准、内容规范、上传指南/切片的评审，同时增加一次不局限于上传指南数据的通用内容巡检。最终两次评审的问题汇总显示，并在同一章节下按来源分为“指南评审”和“通用巡检”两组，继续沿用现有问题卡片、确认、排除、编辑、定位等交互。

## 范围

包含：

- 每个章节运行两类 AI 评审：
  - `guideline_review`：现有基于质量标准、内容规范、参考数据源、重点指南和确认切片的评审。
  - `common_review`：新增通用巡检，只基于疾病名、章节标题、章节原文和词条大纲，检查错别字、重复、表达不合理、前后矛盾、药物用法用量等常见问题。
- 后端返回统一的 `SectionIssue` 列表，并给每条问题标记来源。
- 前端在每个章节结果中按来源分组展示问题。
- 章节顶部的问题数量、维度统计、后续 AI 整合继续把两类问题都纳入。

不包含：

- 自动修正文稿。
- 为通用巡检接入外部医学数据库或联网检索。
- 改变参考文献审核、指南切片确认、AI 整合接口形态。
- 让 AI 对无证据的复杂指南争议下确定性结论。

## 数据模型

扩展后端 `SectionIssue`：

```py
review_source: str = "guideline_review"
```

推荐取值：

- `guideline_review`
- `common_review`

扩展前端 `SectionIssue`：

```ts
review_source?: 'guideline_review' | 'common_review'
```

兼容规则：

- 旧数据没有 `review_source` 时按 `guideline_review` 处理。
- 人工新增问题如果暂时没有来源，也按 `guideline_review` 展示，后续可以单独扩展 `manual`，本期不做。
- `AiIntegrationLinkedIssue` 保留该字段，使 AI 整合历史能追溯问题来源。

## 后端流程

### 指南评审

现有 `analyze_section()` 中的流程基本保留：

1. 使用质量标准、内容规范、参考数据源、重点指南或确认切片生成首轮问题。
2. 对首轮问题进行二次核验。
3. 保留现有误报过滤器、指南依据清洗、原文定位逻辑。
4. 将最终问题统一标记为 `review_source="guideline_review"`。

### 通用巡检

新增 `_review_common_content_issues()`。

输入：

- `disease`
- `section`
- `article_outline`

不输入：

- 上传指南全文
- 重点指南
- 确认切片
- 内容质量审评标准全文
- 内容要求规范全文

Prompt 关注：

- 错别字、别字、明显病句、标点或单位书写异常。
- 内容重复、段落重复、同一观点反复出现。
- 前后矛盾，例如同一章节对适应证、禁忌证、年龄、人群、频次等描述不一致。
- 表达不清或临床表达不合理。
- 药物名称、用法、用量、给药频次、疗程、单位等明显风险。
- 只报告能从章节原文定位的问题。

严格约束：

- 不得把上传指南缺失当作问题，因为通用巡检不看上传指南。
- 不得因为“没有看到指南依据”而报告内容缺失。
- 对复杂医学争议、指南版本差异、需要最新证据才能判断的问题，只能提示“建议人工核对”，不得输出为确定错误。
- `accuracy`、`style`、`outdated`、`structure` 问题必须提供原文 `anchors`。
- 药物用法用量问题必须在 `description` 中说明风险点，并在 `examples` 中保留原文片段；若只是疑似问题，严重程度不超过 `medium`。

输出仍使用 `SectionIssue` schema：

- 错别字、重复、表达问题 -> `style`
- 用法用量、医学常识、前后矛盾 -> `accuracy`
- 明显时效性问题 -> `outdated`
- 内容位置或结构造成重复/误解 -> `structure`

通用巡检完成后，所有问题标记 `review_source="common_review"`。

### 合并与过滤

两类结果不强行混成一组后再丢失来源。

推荐流程：

1. 指南评审内部合并和核验，得到 `guideline_issues`。
2. 通用巡检内部最多输出有限数量问题，得到 `common_issues`。
3. 对两组问题分别运行现有定位和误报过滤。
4. 做一次跨来源轻量去重：
   - 如果两条问题原文 anchor 高度相同且描述同一错误，优先保留 `guideline_review` 问题，因为它可能带有指南依据。
   - 如果通用巡检发现的是更具体的错别字、重复或用量风险，且指南评审未覆盖，则保留。
5. 返回 `guideline_issues + common_issues`，不改变 `SectionAnalysis.issues` 的外层形态。

`verification_summary` 改为综合摘要：

```text
指南评审：核验后保留 X 项。
通用巡检：补充发现 Y 项。
合并去重：删除 Z 项重复问题，最终保留 N 项。
```

## 前端展示

每个章节仍使用现有左右分栏和问题卡片。

右侧问题区调整为：

- 先显示“指南评审”分组。
- 再显示“通用巡检”分组。
- 每组内继续复用现有问题卡片，包括优先级、类型、维度、扣分、定位、确认、排除、恢复、编辑、指南依据展示。

显示规则：

- 分组标题显示有效问题数和总问题数，例如“指南评审 3 项”“通用巡检 2 项”。
- 某组没有问题时可以显示一行轻量空状态，例如“未发现通用巡检问题”。
- 章节标题处的总数仍统计所有非 `rejected` 问题。
- 维度统计仍统计所有非 `rejected` 问题。
- `guideline_evidence` 仍只在 `missing_content`、`accuracy`、`outdated` 且有必要时展示；通用巡检的 `accuracy` 若没有指南依据，不显示“请重新分析该章节”的警告，避免误导。

## AI 整合兼容

质量评审问题带入 AI 整合时，两类问题都可进入候选列表。

任务文本中增加来源标签：

```text
【来源】指南评审 / 通用巡检
```

生成要求保持不变：AI 整合仍需要结合用户选择的参考文献修订内容，通用巡检的问题只作为待解决编辑/医学风险点，不自动等同于已有证据结论。

## 错误处理

- 如果指南评审失败，章节仍按当前行为显示失败，不单独返回通用巡检结果，避免用户误以为完整评审已完成。
- 如果通用巡检失败，但指南评审成功，可以返回指南评审结果，并在 `verification_summary` 中提示“通用巡检失败，建议重新分析该章节”。
- 如果通用巡检返回非法 JSON，沿用 `generate_json` 的修复机制；修复失败后按上一条降级。
- 如果旧历史数据缺少 `review_source`，前端默认归入“指南评审”。

## 测试

后端：

- `SectionIssue` 默认 `review_source` 为 `guideline_review`。
- `analyze_section()` 会调用通用巡检，并将其结果标记为 `common_review`。
- 通用巡检 prompt 不包含上传参考数据源、重点指南、质量标准全文或内容规范全文。
- 通用巡检失败时仍返回指南评审结果，并写入摘要提示。
- 跨来源重复问题优先保留带指南依据的 `guideline_review` 问题。

前端：

- 无 `review_source` 的旧问题归入“指南评审”。
- 两组问题按来源分组展示，并复用确认、排除、恢复、编辑、定位操作。
- 章节总问题数、维度统计包含两组非 `rejected` 问题。
- 通用巡检的 `accuracy` 问题没有 `guideline_evidence` 时，不显示“未提供指南原文依据，请重新分析该章节”。
- AI 整合任务文本包含问题来源标签。

## 实施顺序

1. 扩展后端和前端 `SectionIssue` 类型，保持默认兼容。
2. 新增通用巡检 prompt 与 `_review_common_content_issues()`。
3. 在 `analyze_section()` 中接入第二次评审、来源标记、摘要合并和降级处理。
4. 增加跨来源轻量去重。
5. 前端按 `review_source` 分组渲染问题卡片。
6. 调整指南依据警告逻辑，避免误伤通用巡检问题。
7. 更新 AI 整合问题文本来源标签。
8. 补充后端和前端测试。
