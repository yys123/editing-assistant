# 内容质量评审与 AI 整合连通设计

## 目标

让“内容质量评审”中已经确认的问题，能够直接作为 AI 整合的任务输入，减少编辑复制粘贴和重新组织需求的成本。AI 整合生成的回答应能追溯到它要解决的评审问题。

## 范围

本期只打通“内容质量评审 -> AI 整合”，不自动改写原词条、不自动替换生成稿件。

包含：

- 在 AI 整合页展示可选择的质量评审问题。
- 支持按问题或按章节带入 AI 整合。
- 根据所选问题自动生成可编辑的用户要求。
- 自动选择相关原词条章节。
- AI 整合历史记录保存关联的问题信息。

不包含：

- 自动回写到原文。
- 自动判断问题是否已解决。
- 多轮任务状态流转。

## 用户流程

1. 用户在“内容质量评审”中确认、排除或编辑问题。
2. 用户进入“AI 整合”。
3. AI 整合页显示“来自质量评审的问题”面板。
4. 默认只纳入 `confirmed` 和 `added` 问题；`ai` 状态的问题可显示但不默认勾选；`rejected` 不显示。
5. 用户可按章节展开问题，勾选若干问题。
6. 用户点击“生成整合要求”，系统把所选问题转换为输入框中的任务文本。
7. 系统自动把原词条内容范围切换为“按章节”，并选中相关章节。
8. 用户可继续编辑任务、选择参考文献、设置重点指南，然后发送。
9. 生成后的问答记录保存本次关联的问题摘要。

## 数据模型

新增前端类型：

```ts
export interface AiIntegrationLinkedIssue {
  id: string
  section_id: string
  section_heading: string
  issue_type: SectionIssue['issue_type']
  severity: SectionIssue['severity']
  description: string
  examples: string[]
  reviewer_note?: string
  anchors?: IssueAnchor[]
  guideline_evidence?: GuidelineEvidence[]
}
```

扩展 `AiIntegrationRecord`：

```ts
linkedIssues?: AiIntegrationLinkedIssue[]
```

后端 `AiIntegrationRequest` 暂不强制新增字段。第一期可把问题内容合成到 `user_request`，保持接口兼容。若后续要让后端专门理解问题结构，再增加 `linked_issues` 字段。

## 任务文本生成

系统根据所选问题生成一段可编辑文本：

```text
请根据以下内容质量评审问题，结合原词条内容和已选参考文献，生成可直接用于修订的整合内容。

要求：
1. 优先解决高优先级和准确性/过时类问题；
2. 保留原词条中正确且仍适用的内容；
3. 对新增或修订的事实性内容标注引用；
4. 若证据不足，请明确说明哪些问题暂不能解决；
5. 最后简要列出“已解决的问题”和“仍需人工确认的问题”。

待解决问题：
1. 【章节】...
   【类型】...
   【优先级】...
   【问题】...
   【原文例句】...
   【人工备注】...
   【指南依据】...
```

## UI 设计

AI 整合页新增一个“质量评审问题”区域，放在输入框下方或原词条内容选择上方。

控件：

- 章节分组列表。
- 每个问题一行复选框，显示严重程度、问题类型、描述摘要。
- 操作按钮：`选择高优先问题`、`选择本章节`、`清空`、`生成整合要求`。
- 已选数量提示。

历史记录：

- 问答记录标题仍以用户问题命名。
- 展开后显示“关联评审问题 X 个”。
- 可以折叠查看关联问题摘要。

## 关键规则

- 默认只使用 `confirmed` 和 `added` 问题。
- 生成任务时保留人工修改后的问题描述和备注。
- 若问题有 anchors/examples，应带入作为原文定位依据。
- 若问题有 guideline_evidence，应作为“评审依据”带入，但仍要求 AI 最终引用所选参考文献中的可点击引用标记。
- 自动选择原词条章节时，只选中这些问题所属章节；若没有 parsedArticle，则退回全文。

## 测试

前端：

- 从 `SectionAnalysis[]` 提取可带入问题，排除 rejected。
- 所选问题生成任务文本。
- 所选问题自动映射到章节 id。
- AI 整合历史保存并展示 linkedIssues。

后端：

- 第一阶段无需新增接口测试；已有 AI 整合接口保持兼容。
- 若后续新增 `linked_issues` 字段，再补请求序列化和 prompt 测试。

3. 修改 `StepAiIntegration`，增加质量评审问题面板。
4. 扩展 AI 整合历史记录展示。
5. 增加前端测试并跑构建。
